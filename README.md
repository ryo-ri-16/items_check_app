# items_check_app
お出かけ時に忘れ物がないかを念入りにチェックするアプリです。

## サービス概要
出かける際に必要なものがなんなのかをチェックします。
何月何日何時、あるいは毎週何曜日にチェックするかの設定を行い、通知をさせることで忘れないようにします。
ただリストを見るだけではなく、実際に各項目をチェックするという念入りな確認。

## このサービスへの思い・作りたい理由
これまで生きてきて、学校への登校や旅行などで持ち物を忘れるということは数多くありましたし、大人になった今でも帰省や旅行を行う際に、バッグを持つのを忘れたという経験がありました。
やはりミスを完全に無くすというのは難しく、そのため事前に細かい確認が必要だと思いました。
なお、念入りにチェックするという発想は私が工場で働いている際に指差呼称で保護具や左右の確認をしていることから来ました。実際に対象のものを見て確認し、全てあることを確定させてからアクションを起こすことが重要だと考え、そのような仕様にしようと思いました。

## ユーザー層について
誰でも利用できるようなシンプルなアプリではありますが、私自身が使いたいかという感覚や手っ取り早い、通知などを活用するという点でいえば、日常的な場面で使うことを想定していこうと考えています。学校や職場であれば登校、出勤する際に日にちによって持ってくる教科書や書類などが違うというケースがあったり、ほぼ毎日持っていくようなものであっても些細なことで入れ忘れるということがあります。またプライベートであっても親族や友人と遊びに行く際に何を持っていくかや、病院や銀行、役所などに何らかの手続きをするために保険証や通帳、マイナンバーカードなどが必要になることがあったりと外出をする時に何を持っていくかというのは生活において非常に重要なことであります。私は外出自体は毎日行いますが、外出を一日に二度以上行うのはあまり好まず、休日であれば外食、買い物、必要であれば手続きは一度にまとめます。それでもし必要なものが手元になかった場合はまた後日にするか一度家に帰って取ってくる必要があり非常に面倒です。これが学校や仕事であれば家に帰るという手段すら取りづらくなります。
そこで事前に何が必要であるかをリストにまとめておき、根本的に忘れないようにします。そこから更に実際にチェックを行うという作業を行うことで実際に持っていることを確認してから外出を行うということですね。
ここまでまとめてどういう層に向いているかというと私自身がそうであるように高校生~社会人あたりの段階的にではありますが親から本格的に自立しだしたあたりの人々でしょうか。できることが増えたということはやることも増えていくということでスケジュールやそのスケジュールに必要なものの管理というものは難しくなると思われます。そこで各々のスケジュールに対応できるよう気軽にそれに必要なものが何でかつそれがあるのかどうかというのを確認することで些細なミスを無くしていきます。若干ズレている気がしますが作業においてハインリッヒの法則と言い、些細なミス200回の中から29回のミス、そして致命的なミスが1回起こるということで、小さいミスをなくすことで人生において致命的なミスが発生しないようにするということは重要だと思います。

## サービスの利用イメージ
できるだけシンプルでわかりやすいUIにして、お手軽に使えるようにするよう考えています。
具体的には最小限の動作でリスト作成からチェックまで持っていけるとか、視覚的なわかりやすさ、例えば色の数を少なくすると一画面あたりの情報量は少なめにするとかですかね。また、あくまでリスト作成とその持ち物チェックに特化させることでやりたいことをハッキリとさせます。あとはボタンを大きくしてスマホでの操作がやりやすくするなども考えられます。ただし、ただの手抜きということはなく細かいオプションなどはボタンを押すとバーが出るとか設定ページにて必要か不要かどうかを確認するなどで必要に応じて利用していくような感じです。技術面で言えば本リリースではREACTで楽観的更新やローディングの最小化などフロントエンドをよく活用していきたいですね。
簡単な機能だけで作る場合、数にもよりますが1分にも満たない時間で作ることができるでしょう。
出かける時や場合によっては屋外でも使えることを考えるとアクションの3~５分前あたりに確認するような形でしょうか。
また、日常的な要件でなく、重要な要件の場合は本番の通知だけでなくその10分前や30分前にも場合によっては通知するようにもするべきかもしれません。
誤作動を起こさないためにリストの編集・削除は作成時と平常時に限り、チェック中ではできないようにします。

## サービスの差別化ポイント・推しポイント
既存の旅行用の持ち物アプリや日常用のアプリ、将来的な買い物予定アプリなどを参考として軽く触ってみたりしました。
具体的な差別化ポイントとしては持ち物管理系のアプリは多岐にわたるカテゴリとその中にテンプレートを用意し、そこに自分のものなども入れているように見えました。こちらのやり方としては「段階的機能」として、一旦はリストの作成とアイテムの追加・編集、カテゴリ分けやチェック機能と簡単に利用できることを(初回での説明も込みで)表現することでしょうか。上でも触れた必要に応じてのリマインダー設定にメモやコメントや画像添付、優先度の管理、更にそこから最終段階で使用頻度や共有機能など最初はとにかくユーザーに最小限の機能を提供するという形にします。旅行系のアプリで見た場合は、当然旅行そのものが要点であるため目的地の設定からの交通手段や人数などの部分があったり、よりシンプルなものでも「一つの旅行」に固まっていることが多いように見えました。私としては移動中で忘れた経験もあったようにどのような時でも必要だと思ったのであれば気軽にチェックをするというのを見るため、その旅行でも出かける時、移動中、宿泊中、帰る時など複数回使えるようなやり方が望ましいと考えています。また、持ち物管理や旅行の折衷案として、一回きりの使用にするか日常的に使用するかの取捨選択ができるような形にもしたいですね。
やっていることはtodo系のそれに近いですが、持ち物チェックに特化している点やシンプルな作り、通知を活用する点では時計アプリに近いかもしれません。
あとはリストを見るだけでなく実際にチェックを行うという現場作業から得た思想や状態に応じてできることを制限する状態管理。
日常的なものにするという観点でいえばホーム画面を作った上でそこにその日にはこの時間にこのリストのチェックがあるというのを表示したりとか、設定した土地の天気を表示することで傘が必要かどうかを確認してもらったりするという点が考えられますか。あとは優先度を設定することで特に急いでいるものの確認をできるようにする、特によく使うようなアイテムをあらかじめテンプレートとして用意したりお気に入りで自ら設定する、リスト自体もコピーができるようにすることなどが考えられます。
よく使うリストに関してはそれを可視化できるように最後に使った順や使用数順などのようなソート機能なども思い付きますね。

## 機能候補
### MVPリリース
SPA系のページ構造
リスト一覧ページ:リスト表示、新規作成(PCのフォルダに近い感覚でリストと共存可能なカテゴリーも作る)、フィルタ(50音順や、最後の利用した順番、作成順など)
リスト詳細ページ:持ち物表示、編集(持ち物の追加、削除)、チェック開始ボタン
チェックページ:チェックボックスによりチェック、何個チェックを入れているかの進捗、全てチェックした後の完了ボタン

### 本リリース
設定画面: テーマ、言語、振動、スワイプ操作、天気APIや位置情報を利用するか、通知のリマインダーなど
ホームページ: カレンダーと天気情報、今日のリストや後日のもの。
テンプレート
共有
