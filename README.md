# items_check_app
お出かけ時に忘れ物がないかを念入りにチェックするアプリです。

## サービス概要
出かける際に必要なものがなんなのかをチェックします。
何月何日何時、あるいは毎週何曜日にチェックするかの設定を行い、通知をさせることで忘れないようにします。
ただリストを見るだけではなく、実際に各項目をチェックするという念入りな確認。

## このサービスへの思い・作りたい理由
これまで生きてきて、学校への登校や旅行などで持ち物を忘れるということは数多くありましたし、大人になった今でも帰省や旅行を行う際に、バッグを持つのを忘れたという経験がありました。
やはりミスを完全に無くすというのは難しく、そのため事前に細かい確認が必要だと思いました。
なお、念入りにチェックするという発想は私が工場で働いている際に指差呼称で保護具や左右の確認をしていることから来ました。実際に対象のものを見て確認し、全てあることを確定させてからアクションを起こすことが重要だと考え、そのような仕様にしようと思いました。

## ユーザー層について
誰でも利用できるようなシンプルなアプリではありますが、これが特に必要になりそうな層といえば、「旅行・出張など遠出をすることが多い人」や「幼少な子供を持つ親」でしょうか。
理由としては取り扱う荷物の数が多くなることが予想されるからですね。私は普段はあまり遠出はしませんが実家から離れた場所で生活をしているため、帰省の準備が忙しいことはよくわかります。また上記の通り、乗り物内に持ち物を忘れた経験があるなど、家以外でも確認が欲しい場面はあります。
子育て世代の場合も複数人の場合は誰が何を持つべきなのかという把握するのは難しく、小学生であれば何曜日になんの授業があるかでどの教科書を入れるべきかというのを親子で理解しているのは必要だと思います。あとは買い物にも応用できますね。

## サービスの利用イメージ
できるだけシンプルでわかりやすいUIにして、お手軽に使えるようにするよう考えています。
出かける時や場合によっては屋外でも使えることを考えるとアクションの3~５分前あたりに確認するような形でしょうか。
また、日常的な要件でなく、重要な要件の場合は本番の通知だけでなくその10分前や30分前にも場合によっては通知するようにもするべきかもしれません。
誤作動を起こさないためにリストの編集・削除は作成時と平常時に限り、チェック中ではできないようにします。

## ユーザーの獲得について
現状ではまるでありませんが、強いていうならSNSでRUNTEQの方々に発信することでしょうか。一応X(twitter)にはもう一つそれなりにフォロワーの多いアカウントは持っていますが、匿名でやっているためあまり使いたくはないと考えています。
割と家庭持ちの人がいたり、RUNTEQ_pathでは関東以外の地方から来ている人も結構いたことを考えると一定の需要はあるかと。

## サービスの差別化ポイント・推しポイント
やっていることはtodo系のそれに近いですが、持ち物チェックに特化している点やシンプルな作り、通知を活用する点では時計アプリに近いかもしれません。
あとはリストを見るだけでなく実際にチェックを行うという現場作業から得た思想や状態に応じてできることを制限する状態管理。

## 機能候補
### MVPリリース
SPA系のページ構造
リスト一覧ページ:リスト表示、新規作成(PCのフォルダに近い感覚でリストと共存可能なカテゴリーも作る)、フィルタ(50音順や、最後の利用した順番、作成順など)
リスト詳細ページ:持ち物表示、編集(持ち物の追加、削除)、チェック開始ボタン
チェックページ:チェックボックスによりチェック、何個チェックを入れているかの進捗、全てチェックした後の完了ボタン

### 本リリース
設定画面: テーマ、言語、振動、スワイプ操作、天気APIや位置情報を利用するか、通知のリマインダーなど
ホームページ: カレンダーと天気情報、今日のリストや後日のもの。
テンプレート
共有
